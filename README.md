## 🎯 What it does

EduBob takes a brand-new learner from "what's a variable?" to "I just shipped a small project" through a fully personalized, AI-driven loop:

1. **🎯 Dynamic Onboarding** — 3 multiple-choice questions, generated by watsonx.ai based on the learner's previous answers
2. **📚 Personalized Curriculum** — A 5-topic Python learning path generated specifically for the learner's interests (web / automation / data / AI / games) and skill level
3. **📖 Per-topic loop** — AI-generated lesson + quiz + Monaco-editor coding challenge + AI code review
4. **🔄 Live adaptation** — After each topic, watsonx.ai analyzes the learner's quiz score, code score, and time-on-task, then re-shapes the next topic's depth, examples, and tone
5. **🏆 Capstone project** — Once topics are complete, watsonx generates a personalized capstone project that combines everything into one cohesive build

### What makes it different

- **Genuinely adaptive** — the second topic looks visibly different from the first if the learner struggled
- **Visible AI work** — sidebar AI Activity Log shows exactly what watsonx.ai is doing in real time
- **Personalization meter** — learner can see how much the system has learned about them
- **Transparent reasoning** — "Why this curriculum?" expander shows the profile that drove the choices

## 🤝 For my partner — testing this in 10 minutes

Read this section if you're testing the app, not building it.

### Step 1 — Clone & install

```bash
git clone <repo-url>
cd EduBob/edubob_v2
pip install -r requirements.txt
```

### Step 2 — Get watsonx.ai credentials

You need three values. **Use the IBM-provided hackathon sandbox if you have it** — it's pre-configured, no setup pain.

1. **WATSONX_API_KEY** → https://cloud.ibm.com/iam/apikeys → "Create" → copy immediately (only shown once)
2. **WATSONX_PROJECT_ID** → https://dataplatform.cloud.ibm.com/wx/home → open your project (e.g., "watsonx Hackathon Sandbox") → Manage → General → copy Project ID
3. **WATSONX_URL** → use `https://us-south.ml.cloud.ibm.com` if your project is in Dallas

### Step 3 — Configure `.env`

```bash
cp .env.example .env
# Open .env and paste your three values
```

Your `.env` should look like:

```env
WATSONX_API_KEY=abc123...
WATSONX_PROJECT_ID=xyz789...
WATSONX_URL=https://us-south.ml.cloud.ibm.com
```

### Step 4 — Verify everything works

```bash
python test_setup.py
```

You should see four ✅s:
- ✓ PASS: Package imports
- ✓ PASS: Environment config
- ✓ PASS: Core modules
- ✓ PASS: Watsonx.ai connection

If watsonx.ai connection fails, the most common causes:
- Wrong project ID (paste from the wrong project)
- Forgot to associate the watsonx.ai Runtime service to the project (Manage → Services & integrations → Associate)
- Wrong region in WATSONX_URL

### Step 5 — Run the app

```bash
python -m streamlit run app.py
```

> **Why `python -m streamlit` instead of just `streamlit`?**
> The `streamlit` CLI may not be in your PATH after pip install. The `python -m` form always works.

The app opens at `http://localhost:8501` automatically.

### Step 6 — Smoke test (5 minutes)

Run through the full flow at least once before reporting bugs:

1. **Home page** — Click "🚀 Start Learning"
2. **Onboarding** — Answer 3 questions. Try: *"Beginner / Websites and web apps / Hands-on examples"*
3. **Curriculum** — You should see:
   - A track title that mentions web/web-curious
   - 5 topics with titles that reference URLs, JSON, web data
   - Personalization meter showing some percentage
   - "Why this curriculum?" expander works
4. **Topic 1** — Click "Start Topic 1":
   - Lesson appears (streams in char-by-char on first reveal)
   - Switch to Quiz tab → answer 3 questions → submit
   - Switch to Challenge tab → use the Monaco editor (dark theme, syntax highlighting) → click "Run & Submit"
   - You should see test results + AI feedback + 🎉 + balloons
5. **Topic 2** — Click "Back to Curriculum" → click Topic 2:
   - **Adaptation banner** should appear at the top ("🎯 Adapted for you...")
   - Lesson should be visibly tailored
6. **Capstone** — From any post-topic-1 page, click "🚀 Preview Your Capstone Project":
   - Personalized capstone should generate (~10s wait)
   - Snow effect plays
   - "Generate a different project" button works

### What to look for in the sidebar (always visible)

- **Progress meter** — completed topics
- **🧠 AI Activity Log** — shows live record of every watsonx.ai call with timestamp + token count
- **🔄 Reset Session** — clears all state and starts over

### Common things to flag

| Symptom | Likely cause |
|---|---|
| "Same curriculum every time" | watsonx returning malformed JSON; fallback (per-interest) is firing — still varied |
| Spinner runs forever | Connection timeout or watsonx auth error — check terminal logs |
| Onboarding shows the same question repeatedly | Same as above — fallback firing |
| Code editor shows no syntax highlighting | `streamlit-ace` not installed; rerun `pip install -r requirements.txt` |
| Tab "Take Quiz →" button doesn't switch tabs | Known: Streamlit limitation — user must click the tab manually. Demo voiceover covers this. |

## 🔧 Tech stack

| Layer | Tech | Notes |
|---|---|---|
| **Frontend** | Streamlit + custom CSS | Multipage app, theme via `.streamlit/config.toml` |
| **Code editor** | `streamlit-ace` | Monaco-style editor with monokai theme + line numbers |
| **Runtime AI** | IBM watsonx.ai (`meta-llama/llama-3-3-70b-instruct`) | Powers all 6 product features at runtime |
| **Code execution** | Python `exec()` + `multiprocessing.Process` | Restricted builtins, 5s timeout, Windows-compatible |
| **State** | `streamlit.session_state` | No DB, no persistence — refresh resets the session |
| **Dev assistant** | IBM Bob | Built the entire codebase; sessions exported to `bob_sessions/` |

### About the model choice

We initially planned for `ibm/granite-3-8b-instruct`, but it's not available in the hackathon sandbox's curated model list. We tried `ibm/granite-4-h-small` (returned empty/garbage output for our prompts), then settled on `meta-llama/llama-3-3-70b-instruct` — also served via watsonx.ai — which produces reliable structured JSON for our schemas.

The model ID is a one-line config in `core/watsonx_client.py:28`. To swap, change just that line.

### Watsonx.ai usage breakdown

| Feature | Prompt template | Approx. tokens / call |
|---|---|---|
| Generate next onboarding question | `ONBOARDING_NEXT_QUESTION_PROMPT` | ~300 in / ~80 out |
| Synthesize learner profile | `INTEREST_PROFILE_PROMPT` | ~250 in / ~120 out |
| Generate 5-topic curriculum | `CURRICULUM_PROMPT` | ~400 in / ~600 out |
| Generate topic content (lesson + quiz + challenge) | `TOPIC_CONTENT_PROMPT` | ~500 in / ~1200 out |
| Review submitted code | `CODE_FEEDBACK_PROMPT` | ~600 in / ~300 out |
| Compute adaptation directive for next topic | `ADAPTATION_PROMPT` | ~350 in / ~150 out |
| Generate capstone project | `CAPSTONE_PROMPT` | ~400 in / ~400 out |

All prompts are JSON-constrained (the schema is in the prompt itself) and all responses go through `parse_json_response()` which handles markdown fences and stray prose.

## 🧠 How adaptation actually works

This is the differentiating feature — worth understanding if you're testing.

1. Learner completes Topic N (passes the code challenge)
2. App records a `performance_record`: `{quiz_score, quiz_total, code_score, code_passed, time_seconds}`
3. App calls `adaptation.compute_next_directive()` which sends the performance + history to watsonx.ai
4. Watsonx returns an `adaptation_directive`: `{depth, examples_flavor, extra_emphasis, tone}`
5. Directive is stored in `st.session_state.adaptation_history` keyed to Topic N+1
6. When learner opens Topic N+1, the directive is:
   - Pulled from session_state
   - Passed into `TOPIC_CONTENT_PROMPT` so watsonx generates the lesson with that flavor
   - Rendered as a banner at the top of the topic page
   - Reflected in the personalization meter on the curriculum page

If you want to verify adaptation is actually working (not just claimed):
- In Topic 1, deliberately fail one quiz question
- Submit working code anyway
- Click Topic 2 → the banner should say something like *"shallower depth, more encouraging tone"*
- Topic 2's lesson should be visibly easier than what Topic 1 was

## ⚙️ Known limitations (intentional / time-boxed)

- **No persistence** — refresh the browser, lose your progress. By design (POC scope).
- **No auth / multi-user** — single-learner-per-session model.
- **Tab navigation buttons are advisory** — Streamlit can't programmatically switch tabs from a button click. The "Take Quiz →" button shows an info message; user clicks the tab.
- **Sandboxed code execution is demo-grade** — restricted builtins + multiprocessing isolation are sufficient for honest learners but not adversarial ones. Don't deploy to public internet without proper containerization.
- **Latency** — Llama 3.3 70B is slow (~10–30s per call). Mitigated by `st.write_stream` for perceived speed and per-interest fallbacks if generation fails.
